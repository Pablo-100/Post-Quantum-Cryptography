Chapitre 01 — Contexte

Bits, Moore's Law
& l'Informatique Quantique

La 4ème génération des ordinateurs utilise des transistors et des bits (0 ou 1). Gordon Moore (Intel) a observé en 1965 que le nombre de transistors double tous les 2 ans. Mais cette loi atteint ses limites physiques : à l'échelle atomique, l'effet tunnel et les phénomènes quantiques prennent le dessus.

Moore's Law — La formule
2^((Année - 1949) / 2) = Bits └─ (Année-1949) doit être pair Ex: 2025 → 2^((2025-1949)/2) = 2^38 ≈ 274 milliards
⚠ Limite physique imminente

Les transistors atteignent 1-2 nm (taille de quelques atomes). L'effet tunnel fait traverser les électrons les barrières comme des fantômes. La miniaturisation classique est terminée.

Chapitre 02 — Fondamentaux

Qubits &
Mathématiques Quantiques

Notation de Dirac, kets, bras, produit interne, normalisation, sphère de Bloch, registres n-qubits, produit tensoriel et entanglement — tout ce dont tu as besoin.

Ket |ψ⟩ — Vecteur colonne

Un ket représente un état quantique pur (pure state). Les éléments sont des nombres complexes:

|ψ⟩ = [α₀] où αᵢ ∈ ℂ [α₁] [α₂] Base computationnelle: |0⟩ = [1] |1⟩ = [0] [0] [1]
IMPORTANT

Les kets peuvent avoir n'importe quelle dimension (même infinie!). Les kets unitaires (norme=1) sont des états quantiques valides.

Bra ⟨ψ| — Vecteur ligne conjugué
⟨ψ| = [α₀*, α₁*, α₂*] └─ conjugués complexes du ket Si |ψ⟩ = [1/2, i/2, -1/2, i/2]ᵀ ⟨ψ| = [1/2, -i/2, -1/2, -i/2]
Produit Interne ⟨φ|ψ⟩

Donne deux informations essentielles:

Norme: ||ψ|| = √⟨ψ|ψ⟩ On peut normaliser: |ψ̃⟩ = |ψ⟩/||ψ|| Orthogonalité: ⟨φ|ψ⟩ = 0 → orthogonaux Si unitaires + orthogonaux → orthonormaux
VÉRIFICATION EXEMPLE
|ψ⟩ = [1/√2, i/√2]ᵀ ⟨ψ|ψ⟩ = (1/√2)*(1/√2) + (-i/√2)*(i/√2) = 1/2 + 1/2 = 1 ✓
Normalisation — Condition fondamentale
|ψ⟩ = α|0⟩ + β|1⟩ Condition: |α|² + |β|² = 1 Conservation de la probabilité totale!
Définition Formelle du Qubit

Les états d'un système quantique associé à un qubit sont des éléments d'un espace de Hilbert H₂ de dimension 2, engendré par la base computationnelle {|0⟩, |1⟩}:

|ψ⟩ = α|0⟩ + β|1⟩ = (α) avec |α|²+|β|²=1 (β) Probabilités de mesure: P(|0⟩) = |α|² P(|1⟩) = |β|²

La sphère de Bloch représente géométriquement tout état qubit pur:

|ψ⟩ = cos(θ/2)|0⟩ + sin(θ/2)e^(iφ)|1⟩ θ: angle polaire depuis |0⟩ (pôle nord) φ: phase azimutale

Superposition — La vague
SUPERPOSITION D'ÉTATS

Avant mesure: le qubit est dans une superposition des deux états. La mesure cause un collapse irréversible.

Registre Quantique à n Qubits

Un registre de n qubits ∈ espace de Hilbert H_N de dimension N = 2ⁿ:

n-Qubit register: |Ψ⟩ = Σᵢ αᵢ|i⟩ pour i = 0 à N-1 avec Σᵢ |αᵢ|² = 1 (normalisation) Base de N=2ⁿ éléments: |000...0⟩, |000...1⟩, ..., |111...1⟩ En notation décimale: |0⟩, |1⟩, ..., |N-1⟩
Exemple — Registre 2 qubits (N=4)
|ψ⟩ = α₀|00⟩ + α₁|01⟩ + α₂|10⟩ + α₃|11⟩ Exemple valide (α=1/2 pour tous): |ψ⟩ = ½|00⟩ + ½|01⟩ + ½|10⟩ + ½|11⟩ Vérif: (1/2)²×4 = 1 ✓ Autre exemple: |ψ⟩ = (1/√2)|00⟩ + (1/√2)|11⟩ ← Bell state!
Puissance Exponentielle
n=1 → 2¹ = 2 états n=3 → 2³ = 8 états n=10 → 2¹⁰ = 1,024 états n=50 → 2⁵⁰ ≈ 10¹⁵ états n=300 → plus d'états que d'atomes dans l'univers!
CLÉ DU QUANTUM SPEEDUP

Un registre de n qubits traite 2ⁿ valeurs simultanément grâce à la superposition — c'est l'origine du speedup quantique.

Produit Tensoriel ⊗

Pour A ∈ M^(K,N) et B ∈ M^(L,M), le produit tensoriel A ⊗ B ∈ M^(KL,NM):

A ⊗ B = [a₀₀B a₀₁B ··· a₀(N-1)B] [a₁₀B a₁₁B ··· a₁(N-1)B] [ ··· ] [a(K-1)0B ··· a(K-1)(N-1)B]
Exemple — Matrices
X = [0 1] Z = [1 0] [1 0] [0 -1] X⊗Z = [0·Z 1·Z] = [ 0 0 1 0] [1·Z 0·Z] [ 0 0 0 -1] [ 1 0 0 0] [ 0 -1 0 0]
Exemple — Vecteurs d'état
|0⟩⊗|0⟩ = [1]⊗[1] = [1,0,0,0]ᵀ = |00⟩ [0] [0] |+⟩⊗|+⟩ = 1/2(|00⟩+|01⟩+|10⟩+|11⟩) Superposition uniforme de 4 états! |ψ⟩⊗|ψ⟩ = [1] ⊗ [1] = [1,0,0,0]ᵀ [0] [0]
Quantum Entanglement (Intrication)

Si un registre 2 qubits ne peut pas être décomposé en produit tensoriel d'états single-qubit, les états sont dits entanglés. C'est l'un des phénomènes les plus étranges de la physique.

Mesurer q₀ détermine instantanément l'état de q₁ — "spooky action at a distance" (Einstein)

Les 4 États de Bell (maximalement entanglés)
|Φ⁺⟩ = (|00⟩ + |11⟩)/√2
|Φ⁻⟩ = (|00⟩ - |11⟩)/√2
|Ψ⁺⟩ = (|01⟩ + |10⟩)/√2
|Ψ⁻⟩ = (|01⟩ - |10⟩)/√2
BASE ORTHONORMÉE

Ces 4 états forment une base orthonormée complète de H₄. Tout état 2-qubit s'exprime comme combinaison de ces 4.

Test d'Entanglement
Est-ce que |Φ⁺⟩ = (|0⟩+|1⟩)⊗(|0⟩+|1⟩) ? Développement: = ½(|00⟩+|01⟩+|10⟩+|11⟩) ≠ (|00⟩+|11⟩)/√2 ← ENTANGLÉ! ✓ On ne peut PAS factoriser |Φ⁺⟩ en produit de 2 qubits séparés. → Preuve que c'est entanglé.
APPLICATION CRYPTOGRAPHIQUE

L'entanglement est la base du protocole BB84 et de la Quantum Key Distribution (QKD).

Mesure Quantique — Le Collapse
Pour |φ⟩ = α|0⟩ + β|1⟩: Résultat 0: probabilité = |α|² → état collapse vers |0⟩ Résultat 1: probabilité = |β|² → état collapse vers |1⟩ La mesure est irréversible et probabiliste! Pas comme un ordinateur classique.
GÉNÉRALISATION n-QUBITS

Pour |Ψ⟩ = Σ αᵢ|i⟩, la probabilité d'obtenir l'état |i⟩ lors d'une mesure complète est |αᵢ|².

Demo — Mesurer des Qubits

Principes Clés
1
Non-cloning theorem

Impossible de copier un état quantique inconnu. Base de la sécurité QKD.

2
No-signaling theorem

L'entanglement ne permet pas la communication plus rapide que la lumière.

3
Deferred measurement

On peut reporter les mesures à la fin du circuit sans changer le résultat.

Chapitre 03 — Circuits

Quantum Gates
& Circuits

Les portes quantiques sont des transformations unitaires (U†U = I) — elles conservent les normes et sont réversibles. Nombre d'inputs = nombre d'outputs.

COMMENT CRÉER UN ÉTAT DE BELL
H ⊗ I sur |00⟩ → (1/√2)(|0⟩+|1⟩) ⊗ |0⟩ Puis CNOT → (1/√2)(|00⟩+|11⟩) = |Φ⁺⟩ ✓
Toffoli Gate (CCNOT) — Gate 3 Qubits

Le Toffoli Gate flip le qubit target si les deux control qubits sont à 1. C'est une gate universelle pour le calcul classique réversible.

CCNOT: |x,y,z⟩ → |x,y,z⊕(x·y)⟩ Cas: |x=0,y=?,z⟩ → |x,y,z⟩ (rien si x=0) |x=1,y=0,z⟩ → |x,y,z⟩ (rien si y=0) |x=1,y=1,z⟩ → |x,y,z̄⟩ ← flip! si x=y=1 U_Toffoli = block_diag(I₂,I₂,I₂,X₂) [matrice 8×8]
MCX GATES — GÉNÉRALISATION

Les Multi-Controlled X gates généralisent le CNOT/Toffoli à plusieurs controls (remplis=|1⟩, vides=|0⟩). Essentielles pour MODEXP dans Shor's algorithm.

QFT⁻¹ — Inverse Quantum Fourier Transform

La QFT⁻¹ mappe un état dans le domaine fréquentiel vers la base computationnelle. C'est le cœur de Shor's Algorithm pour révéler la période cachée.

QFT⁻¹|x⟩ = (1/√N) Σₖ e^(2πixk/N)|k⟩ où N = 2ⁿ Gate de rotation: Rₖ⁻¹ = [[1,0],[0,e^(2πi/2ᵏ)]] Structure du circuit QFT⁻¹ sur 3 qubits: q₂: H → R₂⁻¹ → R₃⁻¹ q₁: H → R₂⁻¹ q₀: H Puis: SWAP q₀↔q₂
Pourquoi QFT⁻¹ dans Shor?
1
MODEXP

Crée une périodicité cachée r encodée dans les amplitudes quantiques

2
QFT⁻¹

Transforme la périodicité en pics de probabilité aux multiples de 2ⁿ/r

3
Mesure

x̂ ≈ s·2ⁿ/r → fractions continues → trouver r

Complexité
QFT classique (FFT): O(n·2ⁿ) QFT quantique: O(n²) → Speedup exponentiel!

La QFT quantique est exponentiellement plus rapide que la FFT classique — c'est la source principale du speedup de Shor's algorithm.

Chapitre 04 — La Menace

Shor's Algorithm
Brise RSA

RSA repose sur l'impossibilité de factoriser de grands nombres. Shor (1994) a montré qu'un ordinateur quantique peut le faire en temps polynomial — exponentiellement plus rapide que tout algorithme classique.

SOLUTIONS POST-QUANTIQUES

Post-Quantum Cryptography (PQC): Solutions classiques résistantes aux attaques quantiques. Basées sur des problèmes difficiles même pour un QC (lattices, hash, codes). Remplacent RSA/ECC sans matériel quantique. Standards NIST 2024: FIPS 203, 204, 205.

Chapitre 07 & 08 — CRYSTALS-Kyber

CRYSTALS-Kyber
ML-KEM, FIPS 203

Le standard KEM post-quantique basé sur le problème MLWE (Module Learning With Errors) sur des réseaux de modules. Résistant même aux ordinateurs quantiques.

Qu'est-ce qu'un Lattice?

Un réseau (lattice) est un ensemble de points formés par des combinaisons entières de vecteurs bases b₁,...,bₙ:

L(B) = {Σ aᵢbᵢ : aᵢ ∈ ℤ} Exemple 2D: tous les points (a·b₁ + b·b₂) pour a,b ∈ ℤ
SVP — SHORTEST VECTOR PROBLEM

Trouver le vecteur le plus court non nul dans le lattice. NP-hard — même pour un ordinateur quantique!

CVP — CLOSEST VECTOR PROBLEM

Étant donné un point cible, trouver le point le plus proche sur le lattice. Plus difficile que SVP. Base de la sécurité Kyber.

Lattice Rotatif — CVP Visualisé

Points bleus = lattice 3D rotatif · Rouge = target CVP · Jaune = vecteur solution

LWE Sans Bruit — Facile
Système A × s = t (résolvable!) A = [[14,15,5,2], s = [0 ] [13,14,14,6], [13] [6,10,13,1], [9 ] [10,4,12,16]] [11] t = A×s = [262, 374, 258, 336]ᵀ → Résolu par élimination gaussienne!
LWE Avec Bruit — Impossible!
Système A × s + e = t (mod q) A (7×4), s = [0,13,9,11]ᵀ e = [1,-1,0,-1,1,0,1]ᵀ ← PETIT bruit q = 17 A×s + e (mod 17) = [8,16,3,12,9,16,3]ᵀ → Impossible à inverser! NP-hard même QC.
MLWE — Module Learning With Errors (version polynomiale)

Dans MLWE, les éléments de la matrice et des vecteurs sont des polynômes dans l'anneau Zq[X]/(Xⁿ+1):

Anneau: Zq[X]/(Xⁿ+1) où q=3329, n=256 Réduction: Xⁿ = -1, X^(n+1) = -X, etc. A(X) = [[A₀,₀(X), ..., A₀,ₖ(X)], ← matrice k×k de polynômes [ ... ], [Aₖ,₀(X), ..., Aₖ,ₖ(X)]] s(X) = [s₀(X), ..., sₖ(X)]ᵀ ← petits coefficients e(X) = [e₀(X), ..., eₖ(X)]ᵀ ← petit bruit t(X) = A(X)·s(X) + e(X) ← clé publique Note: tout est modulo (q, Xⁿ+1)

POURQUOI q = 3329?

3329 est un nombre premier tel que 3329 ≡ 1 (mod 256). Cela permet d'utiliser le Number Theoretic Transform (NTT) pour multiplier efficacement les polynômes en O(n log n).

Protocole KEM — Vue d'ensemble

Un Key Encapsulation Mechanism (KEM) permet à deux parties d'établir une clé secrète partagée sur un canal non sécurisé.

A
Alice — Key Generation
Génère matrice A (publique), choisit secret s et erreur e
t = A×s + e    pk=(A,t), sk=s
↓
Alice publie pk = (A, t)
Garde s strictement secret. Tout le monde peut voir (A, t).
B
Bob — Encapsulation
Encode m, tire r, e₁, e₂ aléatoires et petits
u=Aᵀr+e₁   v=tᵀr+e₂+encode(m)
↓
Bob envoie c = (u, v)
Ciphertext transmis via canal non sécurisé (internet).
A
Alice — Decapsulation
Utilise sa clé privée s pour retrouver m
m' = decode(v - sᵀ·u) = m ✓
Pourquoi ça marche? (Correctness)
v - sᵀ·u = (tᵀ·r + e₂ + encode(m)) - sᵀ·(Aᵀ·r + e₁) = ((As+e)ᵀ·r + e₂ + encode(m)) - sᵀ·Aᵀ·r - sᵀ·e₁ = sᵀAᵀr + eᵀr + e₂ + encode(m) - sᵀAᵀr - sᵀe₁ = encode(m) + [petit bruit] Si bruit petit → décodage correct! Toutes opérations mod (q, 1+xⁿ)
FONCTION D'ENCODAGE
encode(mᵢ) = 0 si mᵢ = 0 encode(mᵢ) = ⌊q/2⌋ si mᵢ = 1 Pour q=3329: encode(1) = 1664
FONCTION DE DÉCODAGE

Pour chaque coefficient xᵢ reçu:
Si xᵢ plus proche de 0 → bit=0
Si xᵢ plus proche de q/2 → bit=1

Exemple 1 — Version Scalaire (k=2, n=1, q=17)
Paramètres: k=2, n=1, q=17 A = [[6,5],[3,2]] s = [1,2]ᵀ e = [1,1]ᵀ As = [[6,5],[3,2]] × [1,2]ᵀ = [6+10, 3+4]ᵀ = [16, 7]ᵀ t = As + e = [16,7]ᵀ + [1,1]ᵀ = [17,8]ᵀ = [0, 8]ᵀ (mod 17) → Clé publique: pk = (A, t) = (A, [0,8]ᵀ) → Clé privée: sk = s = [1,2]ᵀ ← GARDÉE SECRÈTE!
Bob choisit: m=1, r=[2,1]ᵀ, e₁=[1,0]ᵀ, e₂=1 Encodage du message: encode(m=1) = ⌊q/2⌋ = ⌊17/2⌋ = 8 Calcul de u = Aᵀr + e₁: Aᵀ = [[6,3],[5,2]] Aᵀr = [[6,3],[5,2]] × [2,1]ᵀ = [15, 12]ᵀ u = [15,12]ᵀ + [1,0]ᵀ = [16, 12]ᵀ Calcul de v = tᵀr + e₂ + encode(m): tᵀr = [0,8] × [2,1]ᵀ = 0+8 = 8 v = 8 + 1 + 8 = 17 = 0 (mod 17) Ciphertext: c = (u, v) = ([16,12]ᵀ, 0)
Alice calcule v - sᵀu: sᵀu = [1,2] × [16,12]ᵀ = 16 + 24 = 40 40 mod 17 = 6 (40 = 2×17 + 6) v - sᵀu = 0 - 6 = -6 -6 mod 17 = 11 Décodage — distance à 0 vs distance à q/2=8: dist(11, 0) = 11 dist(11, 8) = |11-8| = 3 ← plus proche de 8! 11 plus proche de q/2=8 → bit = 1 ✓ Alice retrouve m = 1 = ce que Bob a envoyé ✓

Résumé Mathématique de Kyber
SÉCURITÉ

Basée sur la difficulté de MLWE — trouver s dans A×s+e=t est NP-hard même pour un ordinateur quantique sur des lattices de modules.

EFFICACITÉ

Le NTT (Number Theoretic Transform) permet la multiplication polynomiale en O(n log n) au lieu de O(n²). Très performant sur toutes les plateformes.

POURQUOI Xⁿ+1?

Le polynôme réducteur Xⁿ+1 (avec n=256) garantit que l'anneau polynomial est factorisé selon le NTT, permettant une arithmétique efficace et une sécurité optimale.

PARAMÈTRE η

Les coefficients de s et e sont tirés d'une distribution binomiale centrée Bη — petits mais pas trop petits. η₁ pour s, η₂ pour e₁, e₂.
